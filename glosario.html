Glosario

  • Actores
    Personas o sistemas que interactúan con el sistema principal.

  • Análisis de riesgos
    Identificación de problemas potenciales en el proyecto.

  • Arquitectura de software
    Estructura organizacional de los componentes del sistema.

  • Artefactos
    Productos generados durante el desarrollo de software, como prototipos y diagramas.

  • Automatización de procesos
    Uso de tecnología para realizar tareas con menor intervención humana.

  • Benchmarking
    Comparación de un sistema con estándares o competidores.

  • Control de versiones
    Gestión de cambios en el código fuente del sistema.

  • Cronograma
    Programa de actividades con plazos específicos.

  • Cuestionario
    Herramienta de recopilación estructurada de información.

  • Design Thinking
    Enfoque centrado en el usuario para la resolución creativa de problemas.

  • Diagrama de actividad
    Visualización del flujo de actividades en el sistema.

  • Diagrama de casos de uso
    Representación de las interacciones entre usuarios y el sistema.

  • Diagrama de máquina de estados
    Representación de los estados y transiciones en un sistema.

  • Diagrama de secuencia
    Representación del flujo de mensajes entre componentes del sistema.

  • Diagrama Swimlanes
    Representación gráfica de las tareas en un proceso.

  • Diagramas UML
    Herramientas visuales para modelar sistemas de software.

  • Diseño colaborativo
    Proceso de desarrollo que involucra activamente a los usuarios.

  • Documentación técnica
    Registro detallado de los componentes y procesos del sistema.

  • Empatía
    Proceso de entender las necesidades y experiencias del usuario.

  • Entrevista
    Técnica de recopilación de información mediante preguntas a los usuarios.

  • Equipo de desarrollo
    Grupo encargado de diseñar y construir el sistema.

  • Especificación de requerimientos
    Documento que describe detalladamente los requerimientos del sistema.

  • Estándares de desarrollo
    Normas que guían el diseño y construcción del sistema.

  • Estimación de costos
    Cálculo del presupuesto necesario para el desarrollo.

  • Experiencia del usuario (UX)
    Percepción del usuario al interactuar con el sistema.

  • Extensión
    Relación que indica la necesidad de una acción adicional para completar una función.

  • Gestión de proyectos
    Administración de recursos, tiempo y actividades en el desarrollo.

  • Herramientas automatizadas
    Software que facilita tareas del desarrollo.

  • Historias de usuario
    Descripción breve de una funcionalidad desde la perspectiva del usuario.

  • Inclusión
    Relación entre casos de uso que comparten actividades similares.

  • Ingeniería de requerimientos
    Proceso de identificación, análisis y documentación de los requerimientos.

  • Integración de sistemas
    Conexión de diferentes sistemas para operar como una unidad.

  • Iteración
    Repetición de actividades para refinar el desarrollo del sistema.

  • JAD (Joint Application Development)
    Técnica colaborativa para definir requerimientos.

  • Lluvia de ideas
    Método grupal para generar ideas sobre los requerimientos.

  • Manual técnico
    Documento que detalla la operación y mantenimiento del sistema.

  • Metodología ágil
    Enfoque iterativo para el desarrollo de software que prioriza la flexibilidad y colaboración.

  • Metodología tradicional
    Enfoque lineal y secuencial para el desarrollo de software.

  • Modelado físico
    Representación de la interfaz del sistema.

  • Modelado lógico
    Representación de la estructura interna del sistema.

  • Plan de desarrollo
    Estrategia para ejecutar las actividades del proyecto.

  • Planeación de proyectos
    Organización de actividades y recursos para cumplir objetivos.

  • Prototipo
    Versión inicial del sistema para pruebas y validación.

  • Prueba piloto
    Implementación inicial para evaluar el funcionamiento del sistema.

  • RAD (Rapid Application Development)
    Metodología ágil enfocada en la creación rápida de prototipos funcionales.

  • Reglas de negocio
    Directrices que definen el funcionamiento y restricciones del sistema.

  • Requerimientos críticos
    Funcionalidades indispensables para el éxito del sistema.

  • Requerimientos de dominio
    Características específicas del entorno donde se implementará el sistema.

  • Requerimientos de portabilidad
    Capacidad del sistema para operar en diferentes entornos.

  • Requerimientos de rendimiento
    Especificaciones sobre la capacidad del sistema para operar eficientemente.

  • Requerimientos de seguridad
    Medidas para proteger el sistema contra accesos no autorizados.

  • Requerimientos de software
    Necesidades específicas de datos e información que un sistema debe cumplir.

  • Requerimientos de usabilidad
    Condiciones para garantizar que el sistema sea fácil de usar.

  • Requerimientos funcionales
    Especificaciones que describen las funciones que el sistema debe realizar.

  • Requerimientos no funcionales
    Condiciones que el sistema debe cumplir, como rendimiento y seguridad.

  • Requerimientos opcionales
    Funcionalidades adicionales que mejoran el sistema, pero no son esenciales.

  • Requerimientos prioritarios
    Funcionalidades que deben ser atendidas con urgencia.

  • Requisitos de interfaz de usuario
    Especificaciones sobre el diseño y navegación del sistema.

  • Reutilización de software
    Uso de componentes existentes en nuevos desarrollos.

  • Scrum
    Marco de trabajo ágil para la gestión y desarrollo de proyectos.

  • Sistema multiplataforma
    Software que funciona en diferentes sistemas operativos.

  • Sprint
    Ciclo de trabajo en metodologías ágiles como Scrum.

  • Stakeholders
    Personas interesadas o involucradas en el proyecto.

  • Tarjetas CRC
    Herramienta para modelar sistemas orientados a objetos (Clase-Responsabilidad-Colaboración).

  • Tecnología UML
    Conjunto de herramientas estándar para modelado de software.

  • Testeo
    Evaluación del sistema para verificar su funcionalidad.

  • Trazabilidad
    Seguimiento de los requerimientos a lo largo del desarrollo.

  • Validación
    Confirmación de que el sistema cumple con los requerimientos establecidos.

  • Viabilidad técnica
    Evaluación de los recursos tecnológicos para el desarrollo.

  • XP (Programación Extrema)
    Metodología ágil centrada en el desarrollo iterativo y la colaboración constante con el cliente.
